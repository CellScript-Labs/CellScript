# iCKB CellScript Expressibility Matrix

Support levels:

- `NATIVE`: direct, clear CellScript surface.
- `EXPRESSIBLE_WITH_PATTERN`: possible with recurring idiom or model helper.
- `EXPRESSIBLE_WITH_ESCAPE_HATCH`: needs raw CKB/Rust/codegen-level logic.
- `NOT_EXPRESSIBLE`: not currently representable.
- `OUT_OF_SCOPE`: off-chain or external protocol behaviour.

Evidence statuses used in the test strategy column:

- `EXECUTED_DIFFERENTIAL_SUBSET`: selected normalized fixtures have
  original-vs-CellScript CKB VM differential evidence.
- `EXECUTED_HELPER_ONLY`: compiler/helper-level VM or ELF evidence exists, but
  not full original-vs-CellScript protocol evidence.
- `NOT_FULLY_LOWERED`: the semantic surface still depends on explicit helpers,
  benchmark fixtures, or non-executable assumptions.

## Compiler Features Missing For iCKB-Grade Claims

1. Header dep access that binds an input/receipt/deposit to its block header;
   v0.17 has HeaderDep and input-header accumulated-rate helpers, a DAO-field
   equality lineage helper, NervosDAO type-hash classification, 8-byte DAO
   deposit/withdrawal data
   classifiers, RFC0017 epoch-since constructors, and a DAO relative-epoch
   maturity bridge, plus selected original-vs-CellScript CKB VM differential
   execution. Full automatic lineage/redeem lowering is still missing.
2. First-class Script values and lock/type group views beyond the implemented
   `ckb::current_script_hash()` primitive; v0.17 has a compile-time
   current-role primitive, a runtime current-script-hash primitive, a
   `LOAD_SCRIPT` current/output-lock empty-args guard, full 32-byte SourceView
   lock/type hash requirement helpers, SourceView empty-args helpers,
   SourceView 32-byte args Hash requirement helpers, generic SourceView Script
   code_hash/hash_type identity helpers, and xUDT owner-mode type-args
   verifiers for both explicit `[hash, flags]` and
   `[current_script_hash, flags]`.
3. Executable transaction aggregate invariants with computed expressions, not
   only metadata-only proof-plan records. v0.17 now has executable xUDT group
   amount conservation plus minted/burned delta helpers and strict-mode
   bridges for exact
   conservation plus mint/burn `assert_delta` patterns when the matching helper
   is explicitly called. Computed local `u128` add/sub/mul/div/compare values
   are now addressable by helpers, but automatic lowering still does not derive
   full iCKB receipt/deposit/DAO-rate aggregate equations from declarations.
4. Full MetaPoint group accounting and outputs/outputs-data alignment as
   ordinary typed expressions. v0.17 has input OutPoint tx-hash/index helpers,
   a combined full OutPoint binding helper, a pairwise MetaPoint relative
   helper, fixed-distance and i32-data-driven current-script lock/type pair
   cardinality scans, and executable SourceView occupied/unoccupied capacity
   helpers, plus a Limit Order Match master-OutPoint scan helper, but not a
   first-class MetaPoint/OutPoint value plus protocol-specific maps as ordinary
   typed expressions.
5. DAO and xUDT stdlib primitives that fully match deployed script ABIs; v0.17
   covers selected DAO type/data/header and xUDT owner-mode checks, not every
   deployed-script path.
6. First-class checked 256-bit value arithmetic; v0.17 has executable C256
   requirement helpers but not a `u256`/`C256` value type.
7. Additional signed integer widths if future protocol ports need them; v0.17
   covers the iCKB-required `i32` ABI path.
8. Stable diagnostics/error codes for each lowered invariant family.
